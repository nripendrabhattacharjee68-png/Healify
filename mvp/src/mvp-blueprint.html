Helify Blueprint

Paralysis MVP Frontend

Scope: post-stroke hemiparesis only, home rehab with caregiver support, physio oversight.

Open Current App API Contract

MVP Scope Lock

  • Condition: medically stable post-stroke hemiparesis.
  • Exercises: seated knee extension, assisted straight leg raise, ankle dorsiflexion or heel slide.
  • Safety gate: mandatory pre-session check-in before camera.
  • Outcomes to track: adherence, valid reps, pain trend, alert turnaround.

Screen Flow Blueprint

Onboarding Screen

Collect profile once and lock clinical context for all future sessions.

API: POST /patients/onboarding

Acceptance: affected side is stored and reused in session logic.

Safety Check-In Screen

Hard gate. Session cannot proceed if red flag is marked.

API: POST /sessions/{id}/checkin

Acceptance: red flag creates HIGH alert and blocks camera start.

Guided Session Screen

Camera preview + exercise tracker + assisted mode + real-time cue text.

Camera view (pose landmarks overlay)

API: POST /sessions/{id}/exercise-result

Acceptance: logs valid_reps, avg_rom, assisted_reps.

Session Summary Screen

Show clinical usefulness, not just UI stats.

  • Pain pre/post delta and fatigue delta.
  • Total reps, valid reps, average ROM per exercise.
  • Any generated alerts with severity labels.

API: POST /sessions/{id}/finish

Acceptance: summary is persisted, not local-only.

Progress Screen (7 Days)

Compact trend view focused on adherence and safety.

  • Completed sessions by day.
  • Valid rep rate trend.
  • Pain trend and adherence percentage.

API: GET /patients/{id}/progress?days=7

Caregiver Mode Screen

Large controls and minimal friction for assisted sessions.

  • Large valid/invalid rep buttons.
  • Simple assisted toggle.
  • Voice cues in plain language.

Acceptance: caregiver can only act on linked patient session.

Physio Alert Dashboard

Start with alert queue before advanced AI features.

  • Open alerts by severity and time.
  • Resolve action with short note.
  • Quick open patient detail.

API: GET /physio/alerts, POST /physio/alerts/{id}/resolve

Physio Patient Detail

Single page for review and care-plan edits.

  • Patient profile and current plan.
  • Latest alerts + physio notes.
  • 7-day trends and plan update form.

API: GET /physio/patients/{id}, POST /physio/patients/{id}/plan

Implementation Order (Do Not Skip)

  1. Auth + onboarding + consent + profile persistence.
  2. Safety check-in gate and session lifecycle APIs.
  3. Session summary and 7-day progress.
  4. Physio alerts queue and resolve flow.
  5. Caregiver mode and AI rep correction polish.
